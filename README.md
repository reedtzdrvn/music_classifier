# 🎵 GenreScope — классификация музыкальных жанров с объяснением (Grad-CAM)

Веб-приложение для запуска обученных моделей дипломной работы. Пользователь
загружает аудиотрек, дожидается обработки и получает:

- **предсказанный жанр** крупными буквами и уверенность модели;
- **логиты и вероятности по всем 8 классам** (анимированные полосы);
- **мел-спектрограмму** трека — то, что «видит» модель;
- **Grad-CAM объяснение** — подсветку фрагментов спектрограммы, сильнее всего
  повлиявших на решение модели.

Поддерживаются обе обученные модели:

| Модель | Что это | Файл весов |
|--------|---------|-----------|
| **AST** | Audio Spectrogram Transformer (fine-tuned) | `practical/runs/ast/best/model.safetensors` |
| **CNN** | Свёрточная сеть (4 блока, ~4.7 млн параметров) | `practical/runs/cnn/best.pt` |

Классы (FMA-small): `Electronic, Experimental, Folk, Hip-Hop, Instrumental,
International, Pop, Rock`.

## Архитектура

```
┌──────────────┐      /api/analyze      ┌──────────────┐    /predict    ┌────────────────┐
│   frontend   │ ─────────────────────▶ │   backend    │ ─────────────▶ │   inference    │
│ React + nginx│ ◀───────────────────── │  FastAPI шлюз │ ◀───────────── │ FastAPI + Torch │
│   :8080      │        JSON-результат   │    :8000     │   JSON          │ CNN + AST :9000 │
└──────────────┘                         └──────────────┘                 └────────────────┘
                                                                          модели монтируются
                                                                          из practical/runs
```

- **frontend** — React (Vite), собирается и раздаётся через nginx; проксирует `/api` на backend.
- **backend** — тонкий FastAPI-шлюз: приём файла, валидация (формат, размер), проксирование.
- **inference** — «модельный сервер»: загружает CNN и AST один раз при старте,
  считает предсказание, log-Mel-спектрограмму и Grad-CAM.

### Почему не vLLM

В исходном задании сервис инференса планировался на **vLLM**, но vLLM — это движок
для генеративных **языковых** моделей (LLM). Он не умеет загружать и исполнять
`ASTForAudioClassification` или произвольную свёрточную сеть, а Grad-CAM требует
**обратного прохода** (backward), которого vLLM не предоставляет. Поэтому роль
«модельного сервера» здесь выполняет отдельный микросервис на **PyTorch +
Transformers** — он занимает ту же позицию в архитектуре (отдельный сервис за
backend-шлюзом) и легко заменяется.

## Быстрый старт

Требуется установленный **Docker** с **Docker Compose**. Веса моделей должны
лежать в `../practical/runs` относительно этой папки (как в репозитории).

```bash
cd diploma_project
docker compose up --build
```

Затем откройте **http://localhost:8080**.

> Первая сборка скачивает CPU-сборку PyTorch (~200 МБ) и может занять несколько минут.
> Модели **не копируются** в образ — они монтируются как том только для чтения.

Полезные адреса:

- Веб-интерфейс: <http://localhost:8080>
- Swagger backend API: <http://localhost:8000/docs>
- Проверка состояния: <http://localhost:8000/api/health>

Остановить:

```bash
docker compose down
```

## Как работает Grad-CAM

- **CNN.** Хук на последний свёрточный блок (`block4`); веса каналов — глобально
  усреднённые градиенты, карта = `ReLU(Σ wₖ·Aₖ)`. Трек разбивается на 5-секундные
  окна, карты сшиваются по оси времени и выравниваются с полной спектрограммой.
- **AST.** Хук на выход последнего блока трансформера; важность токенов считается
  как Grad-CAM по эмбеддингам (`ReLU(Σ w·A)`), токены-патчи (12×101) разворачиваются
  в частотно-временную сетку, апсемплятся и накладываются на спектрограмму.

## Локальная разработка (без Docker)

Frontend:

```bash
cd frontend
npm install
npm run dev            # http://localhost:5173, проксирует /api на :8000
```

Inference — быстрая проверка логики на одном файле (нужен Python-venv проекта):

```bash
cd inference
set CNN_CKPT=..\..\practical\runs\cnn\best.pt
set AST_DIR=..\..\practical\runs\ast\best
python smoke_test.py path\to\track.mp3
```
